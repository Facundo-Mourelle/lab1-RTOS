### Organización de archivos

Como los programas se compilan directamente utilizando los contenidos en `main.c`, cada desafío esta contenido en una subrama diferente a la de *main*, en donde se ubica el template a utilizar.
Dichas subramas son **"desafioX"**.

### Instalación y ejecución

Para compilar manualmente el proyecto se utiliza *arm-none-eabi-gcc*, y para facilitar la compilación se utiliza la herramienta [bear](https://github.com/rizsotto/Bear) con el comando `bear -- make`
