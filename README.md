### Organización de archivos

Como los programas se compilan directamente utilizando los contenidos en `main.c`, cada desafío esta contenido en una subrama diferente a la de *main*, en donde se ubica el template a utilizar.
Dichas subramas son **"Desafio_X"**.

### Instalación y ejecución

> [!note] Se incluyen los binarios compilados ejecutables, por lo que es posible ejecutar directamente el código en las placas

Para compilar manualmente el proyecto se utiliza *arm-none-eabi-gcc*, y para facilitar la compilación se utiliza la herramienta [https://github.com/rizsotto/Bear](bear) con el comando `bear -- make`
